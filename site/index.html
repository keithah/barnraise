How BarnRaise Works

A simple process that respects everyone involved

1

Request a Feature

Someone needs a new feature in an open source project. They post a "raising" with a description and funding offer. Think of it as saying "I need this barn built."

2

Maintainer Approves

The project maintainer reviews the request. If it fits their vision, they approve it. This ensures only wanted features get built. No unwanted pressure!

3

Developer Volunteers

Developers see approved raisings and volunteer to build them. The maintainer picks the best person for the job. Everyone knows what they're getting into.

4

Money Goes to Escrow

Only after everyone agrees does money move. It's held safely in escrow until the work is done. No risk for anyone.

5

Feature Gets Built

The developer builds the feature with guidance from the maintainer. Progress is visible to everyone. It's collaborative, not transactional.

6

Everyone Wins

Feature is merged, developer gets paid, project gets better, sponsor gets their feature. The community celebrates together!
